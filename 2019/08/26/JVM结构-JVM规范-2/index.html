JVM结构-JVM规范-2

发表于 2019-08-26 | 分类于 Java虚拟机规范-基于JavaSE8 |
字数统计: 902 | 阅读时长 ≈ 3

​ 本章定义了一个抽象的机器，并不会描述任何的JVM的特定实现。

​ 正确的实现一个JVM，你只需要能够正确的读取字节码文件并执行其中定义的操作。具体实现细节并非 JVM规范(本书)的内容，所以它不会也没必要限制 JVM实现者的创造力。例如，运行时数据区的内存结构层次，使用的GC算法，虚拟机指令的内部优化(例如 将字节码翻译为机器码)，这些内容都留给实现者(即 不同的JVM，HotSpot，Taobao JVM等)去具体实现。

字节码文件 格式

​ 可由Java虚拟机执行的编译后的代码使用与硬件和操作系统无关的二进制格式表示，通常(但不一定)存储在文件中，即 字节码文件(class 文件)。字节码文件格式 清晰的定义了 类或接口 的表征，包括一些细节，比如字节顺序，这些在特定平台的目标文件格式中可能被认为是理所当然的。

​ 本书第四章 对此有更详细的数说明。

数据类型

​ 与Java编程语言一样，Java虚拟机操作两种类型:原始类型和引用类型。相应地，原始值和参考值 也可以存储在变量中，或作为参数传递，活由方法返回并操作。

​ Java虚拟机 希望 几乎所有类型检查都在运行时之前完成，通常该检查由编译器完成，而不需要由JVM本身来完成。基本类型的值不需要在运行时确定它们的类型，或者与引用类型的值区分开来，所以它不要被标记，或以其他方式被检查。相反，Java虚拟机的指令集通过 使用对特定类型的值进行操作的指令来区分操作数类型。例如，iadd、ladd、fadd和dadd都是Java虚拟机指令，它们都是求两个数值之和并生成数值结果， 但是每个指定都有各自特定的操作类型：int，long,float 和 double

​ Java虚拟机包含对对象的显式支持。对象要么是动态分配的类实例，要么是数组。对对象的引用被认为拥有Java虚拟机类型引用。引用(reference)类型的值可以看作指向对象的指针。一个对象可能存在多个引用。对象总是通过引用类型的值操作、传递和测试。

原始类型与值

JVM支持的原始数据类型为 数字类型，boolean类型 和 returnAddress 类型(returnAddress type).

数字类型包含 integral 类型 和 浮点类型(floating-point types).

integral类型包括:

  • byte 8位，带符号的二进制补码整数， 默认值为 0
  • short 16位 ,带符号的二进制补码整数，默认值为 0
  • int 32位 ,带符号的二进制补码整数，默认值为 0
  • long 64为位 ,带符号的二进制补码整数，默认值为 0
  • char 16位无符号整数，使用UTF-16编码，其默认值为空码点(‘\u0000’)

浮点数类型包括:

  • float 32位 其值是浮点值集的元素，或者在支持的情况下 为浮点数扩展指数 值集合，其默认值为 正0
  • double 64位 其值是 双精度浮点值集的元素，或者在支持的情况下 为双精度浮点数扩展指数 值集合，其默认值为正0

(题外: float 的范围 大于 long)
