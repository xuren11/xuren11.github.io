Nginx目录结构

/etc/nginx/ : nginx的默认安装目录
/etc/nginx/nginx.conf ： nginx Service服务的 配置入口。该配置文件中设置了 全局配置，例如 工作进程，调试，日志，动态加载模块 ，并且也会引用其他的Nginx配置文件。在默认配置中， 该文件包含了 最高等级的Http 块，即 接下来所述目录中的所有配置文件。
/etc/nginx/conf.d：包含默认的HTTP 服务器配置文件。其中以 .config结束的文件被 包含在最高等级的HTTP块中(如上，即/etc/nginx/nginx.conf中)。最佳实践就是 利用 包含块 ( 即 将配置按类别 放在不同的地方， 在使用时引用配置块)，保持配置文件的简介明了。
/var/log/nginx/：Nginx默认的日志位置。其中含有 access.log 和 error.log文件。access日志文件 记录了每次对 Nginx服务器的请求。error 日志则包含 所有的错误时间 和 调式信息(如果调试模块 开启了的话)

Nginx命令

nginx -h: -h linux环境下的 几乎通用的 显示帮助的命令
nginx -v : nginx版本查看
nginx -V: 显示更详细的版本信息
nginx -t: 测试NGINX配置 （一般用在修改 了配置文件的时候）
nginx -T: 测试 NGINX的配置信息，并打印 有效的配置在屏幕上。该命令用于在 寻求支持的时候。

nigix -s -singnal : -s 表示发送信号给NGINX master进程。你能发送信号 如 stop，quit，reload 和 reopen。 stop 表示马上停止服务。quit表示完成当前所有任务后再停止服务。reload表示 重新加载配置。reopen 表示让NGINX重新打开 日志文件(没测试除开具体是个啥意思……….)

提供静态内容

利用如下NGINX 配置示例 重写位于/etc/nginx/conf.d/default.conf 中默认的HTTP 服务配置：

1.定义了一个 Server 块。告诉NGINX要监听这个 新环境.
2.告诉NGINX指定监听80端口，“default_server” 参数告诉NGINX用该服务器作为 80端口的默认环境。
3.“server_name”定制了主机名，或者说 哪个请求应该被指向该server(和hosts文件里的一样)
(如果配置没有指定该环境作为 default_server,那么只有HTTP请求头直接匹配 server_name 的值，该请求才会被直接定向到该 server) [测试是一直没测试出来的改了这个配置有什么用……学习后再来补]
4.location块定义了基于URL中的路径的配置。这个路径，或者域之后URL的一部分，被称为URI。NGNIX将会完全匹配请求到location块的URI。本例中使用 / 匹配所有请求。
5.root指令展示了NGINX在提供服务时去哪寻找静态文件。在寻找被请求的文件时，请求的URI被追加到root指令值的后面。如果我们提供了URI前缀给location指令，该前缀同样会被包含在 追加路径中。除非我们使用alias指令。
6.index指令提供给NGNIX默认的文件，或者一组待检查的文件， 在URI没有提供更多的信息时(即 没有指定访问那个文件时)
